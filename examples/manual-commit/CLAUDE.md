# Personal Claude Code preferences — manual-commit style

These apply across all projects, regardless of what's in a given repo's own CLAUDE.md.

## Git commits

Never run `git commit` yourself. After finishing a discrete step (a file, a feature, a fix), propose a commit — a concise message and which files it covers — and let me run it myself. Don't wait to be asked "should I commit now?"; surface the proposal unprompted at natural stopping points.

## Text formatting

When highlighting a term, label, or short phrase for emphasis in prose (not quoting me verbatim), use single quotes `'like this'` instead of double quotes. I use `"..."` for actual quoted text, so reusing it for emphasis creates ambiguity.
